Manual de instruccion para usuario tarea 1:

El modulo principal es el llamado Interfaz.py, en el cual al ejecutar el programa se le pide al usuario ingresar contraseña y nombre de usuario, el programa busca las contraseñas y nombres de usuario en el archivo "usuarios.csv", y en el caso de que las ingresadas se encuentren en el archivo dejará al usuario ingresar al sistema. Dependiendo del usuario la interfaz será distinta, si es miembro de ANAF se verá en el display varias opciones, en caso de que sea piloto o jefe de brigada solo se mostraran dos, ya que el programa quedó incompleto. Cabe señalar que el programa es robusto al input del usuario, en caso de que la informacion ingresada no tenga el formato correcto el programa va a imprimir que el formato es incorrecto y que lo vuelva a ingresar, pero no se caerá.
Las funciones implementadas para los usuarios ANAF son: leer las bases de datos y agregar nuevos datos a ellos, en el código se considera que la informacion en el archivo esta desordenado.
Aparte de agregar y leer, el código tiene implementadas las funciones de cerrar sesión, que permite al usuario cambiar de sesión sin tener que reiniciar el programa, y cambio de fecha, que da la posibilidad al usuario de cambiar la hora actual las veces que quiera.

Las funciones mencionadas son las que son visibles para el usuario, pero tambien se crearon otros modulos que tienen funciones especificas, aunque lamentablemente no pudieron ser acopladas al programa principal. En el módulo llamado incendios se implementaron funciones para leer el archivo de "meteorologia.csv" y poder calcular el radio de un incendio conociendo la fecha actual, la funcion ordena los pronosticos climáticos por cercanía física, para posteriormente analizar si es posible que el incendio haya llegado a intersectar el suceso climatico, teniendo en consideracion la fecha actual, en el caso de que sea posible la intersección se revisa si a la hora que el incendio tocó el suceso aún es valido el pronostico, si aún tiene validez se agrega a una lista, luego el programa intersecta todas las fechas posibles, para asi obtener la velocidad de expansión del incendio considerando la suma de los eventos climáticos. Una vez que el programa tiene todas las intersecciones se encarga de sumarlas para asi tener el radio del incendio. Con el radio el programa también es capaz de almacenar en una variable la altura de las nubes, en el caso de que no hayan registros el programa almacena que están a una altura de 2000 metros ya que a esa altura los pilotos pueden volar sin inconvenientes.

Por último se creó el modulo estrategias.py que contiene funciones para que el usuario ANAF pueda elegir una estrategia de extinción (bajo costo, menor tiempo, menores movilizaciones), este modulo no esta completo por lo que no pudo ser implementado a la función principal. El modulo lee el archivo "recursos.csv" y luego escribe un reporte en el archivo de texto "reportes de extincion.txt" a continuación se explica la lógica detras de cada una de las 3 estrategias de extinción.

Estrategia de menores movilizaciones, el programa lee la base de datos y agrega todos los recursos que puedan ir al incendio(con excepcion a los pilotos de aviones o helicopteros cuando las nubes se encuentren bajas) a una lista, y se ordena de mayor a menor dependiendo de la tasa de extincion por la autonomía, luego se va enviando de a uno y revisando si el incendio se ha apagado, si se apagó no se envían más, si continua se envía el siguiente de la lista.

Estrategia de menor tiempo, se agregan a una lista los recursos que tenga menor distancia con el incendio, el primer recurso que llegue al incendio apagará al incendio con su tasa de extincion, una vez que llegue otro recurso al incendio se sumarán las tasas de extincion y se agregará un timer de cuanto tiempo de autonomía le queda al recurso anterior, una vez que el timer llegue a su fin se restará su tasa de extincion a la suma, este proceso se repite hasta que el incendio se extinga completamente.

Estrategia de menor costo, se ordenan los recursos de menor a mayor dependiendo de su costo, y se van enviando al incendio, una vez que el incendio es ha sido extinto se deja de enviar recursos.
